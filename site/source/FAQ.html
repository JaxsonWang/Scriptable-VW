Scriptable 应用

Scriptable 应用收费吗？为什么是英文的？

目前 Scriptable 应用是免费的，作者后续会添加简体中文支持，不过这也没关系，因为用上 Joiner 小组件接触到 Scriptable 核心内容很少，无需担心软件带来心智负担的问题。

运行脚本或者打开软件提示：「Access to contacts is not granted. Grant access from the system Settings.」问题？

1. 请到「设置 - 隐私 - 通讯录」找到 Scriptable 应用赋予权限

2. 请到「设置—通知—scriptable」 打开通知。

Joiner 小组件

Joiner 小组件是否存在安全性问题？

目前 Joiner 小组件代码社区开源，没有后门也没有广告，安全透明，并且编译的代码没有混淆，欢迎大家来审查代码！

为什么我的车辆位置信息一直不出现？

1. 例如首先打开一汽奥迪 - 用车 - 更多功能 - 车辆定位 是否正确显示车辆位置，大众根据自己情况而定。

2. 如果不能，则需要车上的 mib 系统关闭隐私选项，并且开启车辆位置开关

3. 如果还不能请确保车辆是否自带导航功能，不带定位功能被阉割

4. 如果上述描述都正常，请申请高德 key 密钥，后期版本不内置，请自行注册高德 key。

5. 如果上面不管用，可能你的车设备出现问题

出现莫名其妙的红色错误怎么办？

能正常使用小组件提前需要你的车辆管理应用中的用车功能是正常的，即能看到续航、燃料、锁车车窗等状态，如果应用都不能正常显示，那么小组件也无法正常显示。

为什么要准备俩个车辆管理（例如一汽奥迪、上汽大众等）账户

目前大众集团车辆应用支持单点登陆，即设备A登陆账户，如果设备B再登陆会把设备A的账户挤出去。

所以建议再注册一个大众集团车辆的账户，例如：一汽奥迪 - 用车 - 更多功能 - 用户管理 - 添加授权用户。

这个授权用户只用于小组件登陆使用，这样不会出现顶号并且组件莫名其妙报错的问题。

快捷指令

怎么和 Siri 进行交互呢？

目前「FVW Joiner v1.0.0」和「SVW Joiner v1.0.0」第一版本支持，而「FVW Audi Joiner v1.3.0」以上版本包括此版本支持和系统的 Siri 交互，请到脚本的弹窗脚本进行检查版本。

然后进行在下列资源列表下载快捷指令导入就可以使用了。

如果用「Joiner」小组件和「Siri」进行交互需要注意什么？

除了版本限制以外，还要保证「FVW Audi Joiner」、「FVW Joiner」和「SVW Joiner」的脚本名称是不允许更改的，否则快捷指令无法使用。

如果我想根据「Joiner」自定义新的快捷指令，怎么设计？

目前脚本只把车辆的基本信息暴露出来，作为一个词典数据（也叫 JSON 格式数据），目前数据格式如下： 

{
  size: '', // 暂无用处
  seriesName: '', // 车辆名称
  modelShortName: '', // 车辆功率
  vin: '', // 车架号
  plateNo: '', // 车牌号码
  endurance: 0, // 当前车辆续航
  fuelLevel: 0, // 当前燃油/电池容量
  oilLevel: undefined, // 机油信息（如果出现 undefined 则车辆不支持显示）
  mileage: 0, // 当前车辆总里程
  updateDate: '', // 更新日期
  carSimpleLocation: '', // 简洁的地理位置
  carCompleteLocation: '', // 完整的地理位置
  longitude: '', // 车辆纬度
  latitude: '', // 车辆经度
  status: true, // 车辆锁车状态 布尔值 true = 已锁车 false = 未锁车
  doorAndWindow: '', // 车窗状态 布尔值 true = 车门车窗已关闭 false = 车门车窗没有关闭
  myOne: 'xxxx' // 一言
}
